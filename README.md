Windows:
You will need to have python3 installed and available from a CMD terminal. If you wish to compile the program into an .exe to save time, you may do so.
Some modules such as PyAudio are not supported on Windows. You may encounter warnings whilst running the setup.py file, but should any error occur, please notify me.

Linux:
Refer to the advice given when running setup.py, you may be required to install some python packages directly from the apt repository if this is your first time running FiEncrypt.

Compatibility:
FiEncrypt is actively tested on Debian/Ubuntu linux distros (My computers) and has been ported to Windows to expand compatibility. The use of certain GUI and command line print functions can be intensive for lower-end CPU's, and produce weird output when dealing with rapid line position editing and writing. If there is not currently a 'barebones' setting that avoids all the fancy aesthetics, it is absolutely a priority of mine. Any operating system not mentioned is not officially supported, however bug reports will be addressed nonetheless.

About FiEncrypt:
I started FiEncrypt as a pet project around April 2020, as an individual worried by the censorious behaviour of big tech "platforms", I felt I needed to throw my weight behind the free-speech respecting moving in the tech-sphere. While there are great alternative social platforms like Gab, Minds, BitChute, PocketNet, etc... I wanted a program that fulfilled the role of a messenger and file exchange with no potiential for censorship or provider denial attacks. 

The essence of this program is the ability to send messages/files to other users in a unique and entirely decentralised method. With the exception of some other non-essential modules, FiEncrypt does not rely on any first or third party servers. The encryption method is technically mixed ASCII-offset, using various non-linear tweaks to each packet (such as reversing stirng, etc...). These procedures will change as I discover more efficent and sturdy ways of protecting your privacy, and I aim to at some point implement the ability for you, the user, to modify the procedure in such a way that any exploits will be easily rendered obsolete.

Interaction with FiEncrypt may not be the most straightforward process for those less savvy with computers, which is why I enabled the GUI by default (I personally prefer the command line), but do not expect to get the hand-holding that Windows and other software developers offer. 

With a passionate and dedicared community behind it, FiEncrypt can thrive.
